## 一、项目定位

### 1.1 目标用户
- 网络运维工程师（中小企业 IT、集成商现场工程师）
- 网络初学者、考证学员（HCIA/HCIP/CCNA/CCNP）
- 网络专业学生、培训机构

### 1.2 核心价值
**"画图勾选 → 一键生成多厂商命令"** —— 把繁琐的命令行工作图形化，降低多厂商配置门槛。

### 1.3 与原站差异化思路（避免直接侵权 + 形成竞争力）
| 维度 | 原站 qsxwl | 本项目（建议） |
|---|---|---|
| 形态 | Web SaaS | Web SaaS + 离线桌面版 |
| 拓扑画图 | 较基础 | 引入 drawio 级体验 + 自动布局 |
| AI 能力 | 无 | LLM 辅助：自然语言→命令、命令解析、配置审计 |
| 命令解析 | 中文注释 | 中文注释 + 命令对比 + 风险提示 |
| 开放度 | 闭源付费 | 厂商命令模板开源（社区贡献），核心引擎收费 |
| 协作 | 单人 | 多人协作画图、版本管理 |

3. **多厂商命令引擎**：13 厂商（华为/H3C/思科/锐捷/迈普/海康/TP-LINK/烽火/迪普/浪潮/博达/瑞斯康达/神州数码）

5. **会员/支付系统**：套餐选购、兑换码、登录鉴权
6. **附赠资源**：ENSP 实验包、CRT 工具、视频课程

### 2.3 命令覆盖矩阵（待支持功能）
- **交换机**：VLAN（端口/IP/MAC）、IP 路由、端口聚合、DHCP、防私接小路由、远程管理、生成树、IP+端口绑定、静态路由、端口镜像、网段隔离、VRRP+MSTP、OSPF、策略路由
- **路由器**：拨号/固定 IP 上网、端口映射、静态路由、OSPF、IP 黑名单、多运营商分流、远程管理
- **防火墙**：接口区域、域间策略、拨号/固定 IP、端口映射、静态路由、OSPF、远程管理

## 三、技术架构

### 3.1 技术选型
| 层 | 选型 | 理由 |
|---|---|---|
| 前端 | **Vue 3 + Vite + Element Plus** | 国内生态成熟，与原站风格契合 |
| 画图引擎 | **AntV X6** 或 **drawio-mxgraph** | X6 更轻量、可定制；drawio 体验最佳 |
| 状态 | Pinia | Vue 官方推荐 |
| 后端 | **Go (Gin)** 或 **Node.js (NestJS)** | Go 部署简单/性能好；Node 与前端同构 |
| 命令引擎 | **模板引擎 + 规则 DSL**（详见§五） | 多厂商可扩展 |
| DB | MySQL 8 + Redis | 关系数据 + 会话/限流 |
| 文件存储 | 本地或 OSS | 用户拓扑图、生成的配置文件 |
| 支付 | 微信支付 + 支付宝（个人开发者可用虎皮椒/payjs） | 国内主流 |
| 部署 | Docker + Nginx + 阿里云轻量 | 成本可控 |

### 3.2 系统架构图
```
[浏览器]
   │  HTTPS
   ▼
[Nginx]──静态资源
   │
   ▼
[API 网关 / Gin]──┬──[鉴权服务] ── Redis(Session)
```

## 五、命令生成引擎（最核心模块）

### 5.1 设计思路
**特性 (feature) × 厂商 (vendor) → 模板 (template)** 的二维矩阵，模板用占位符渲染。

### 5.2 模板示例（YAML，仅作格式示意）
```yaml
feature: vlan_by_port
vendors:
  huawei:
    template: |
      vlan {{ vlan_id }}
       description {{ desc }}
      interface {{ port }}
       port link-type access
       port default vlan {{ vlan_id }}
    annotations:
      - "vlan {{ vlan_id }}: 创建 VLAN"
      - "port link-type access: 设置端口为接入模式"
  h3c:
    template: |
      vlan {{ vlan_id }}
       description {{ desc }}
      interface {{ port }}
       port link-type access
       port access vlan {{ vlan_id }}
  cisco:
    template: |
      vlan {{ vlan_id }}
       name {{ desc }}
      interface {{ port }}
       switchport mode access
       switchport access vlan {{ vlan_id }}
```

### 5.3 引擎流程
```
拓扑图 JSON  ─┐
参数勾选数据 ─┼─► 参数归一化 ─► 选择厂商模板 ─► 渲染 ─► 拼接注释 ─► 输出
设备型号    ─┘
```

### 5.4 复杂特性策略
- VRRP+MSTP、OSPF 等多设备协同特性 → 在拓扑层面识别"逻辑组"，遍历每台设备调用对应模板
- 提供 **DSL 校验**：参数缺失/冲突预警（例如 VLAN ID 范围、端口名格式）

## 八、风险与合规

### 8.1 法律风险（重要！）
- **不得直接复制**原站文案、图片、视频、CSS 样式表。所有 Hero 标语、卡片配图必须重新设计/原创撰写
- ENSP 模拟器、CRT 等**不能直接打包分发**，应改为"提供官方下载指引"
- 软著最好登记，避免反被诉抄袭
- 备案：ICP 备案 + 公安备案；涉及视频教学需考虑《网络视听许可证》（个人可先用第三方平台嵌入）

### 8.2 命令准确性风险
- 多厂商命令差异极大，**必须有真机/模拟器验证流程**
- 引入"命令模板版本"概念，错命令可快速回滚
- 提供"用户报错反馈"入口，社区共建

### 8.3 商业风险
- 原站价格已极低（终身约 398 元）。纯模仿很难赢，**务必走差异化**（AI / 协作 / 开源生态）
- 初期可考虑"免费 + 高级特性付费"的 freemium 模式获取流量
